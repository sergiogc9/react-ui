# Table

## Introduction

The table components helps is useful to show data in grid.

It's built based on the library `react-table`, hence its documentation should be checked before going further in this documentation:

[React table docs](https://react-table.tanstack.com/docs/overview)

The Table components are the following:

- `Table`: The table wrapper component itself. It is a Box component with `display: block` which encapsulates all the logic together with `react-table`. This component **must wrap the other components**.
- `Table.Content`: The component that renders the table content (headers and rows). It is a Box component which can receive all Box Props **except flexbox related props** because it is not a flex container.
- `Table.HeaderCell`: The component to render as a header column cell. It encapsulates some logic as, for example, showing or hiding the sorting arrows. It should only be used if custom header cells are needed.
- `Table.Cell`: The component to render as a row cell. It basically prints the value of the column for the desired row. This is one of the built-in `Cell` components that are available to use (more info below).
- `Table.Toolbar`: A simple Box component to use as a Toolbar. It should be used above or below the content.
- `Table.TotalResults`: A helpful component to render the total number of rows. Use its `render` prop which receives the number of rows to render the desired content.
- `Table.Pagination`: A plug and play component to show a simple pagination.

As it can be seen in `react-table` docs, the `useTable` hook needs an `options` object and some **plugins**. The Table component can receive these options but the plugins used can not be customized in each Table.

The used plugins inside Table are:

- `useFlexLayout`: Defines the layout.
- `useSortBy`: Helps sorting the table results.
- `useFilters`: Helps filtering the table results.
- `usePagination`: Helps with paginating the results.

## Table component

The Table component can receive any prop from Box component. Also it has the following props:

### `columns`

The columns prop is **required** and it must be an array of [Column options](https://react-table.tanstack.com/docs/api/useTable#column-options). It accepts all properties specified in `react-table` docs plus the added by the available plugins.

Important:

- It must **be memoized**.
- Nested column grouping is supported.
- Each column must **have both Header and Cell attributes**.
- Header attribute can be a string or a Component. If a string is provided, the default `Table.HeaderCell` is used.
- Cell attribute must be a Component. It can be one of the pre-defined (`Table.Cell`, `Table.Cell.Date`, etc...) or a totally customized one. Keep in mind that the returned JSX by this component will be placed inside a Box component.
- By default, each column width **is automatically defined based on all cells value**. See columns width section for further documentation.

Example with three columns:

```ts
type Data = {
	name: string;
	email: string;
	age: number;
};

const columns = React.useMemo<Column<Data>[]>(
	() => [
		{
			id: 'name',
			accessor: 'name',
			Header: 'Name', // Use default Table.HeaderCell,
			Cell: Table.Cell,
			width: 100
		},
		{
			id: 'email',
			accessor: 'email',
			// Example with custom content and style
			Header: props => (
				<Table.HeaderCell {...props} justifyContent="center">
					<Icon icon="info" styling="outlined" />
				</Table.HeaderCell>
			),
			// Example with custom cell
			Cell: props => (
				<Table.Cell.Default {...props} justifyContent="center">
					<TextField value={props.value} />
				</Table.Cell.Default>
			),
			width: 200
		},
		{
			id: 'age',
			accessor: 'age',
			Header: 'Age',
			// Example with date specific pre-defined cell component
			Cell: Table.Cell.Date,
			width: 100
		}
	],
	[]
);
```

### `data`

The data prop is **required** and it must be an array of objects with shape matching the generic type `Data` provided in Columns.

Important:

- It must **be memoized**.
- All accessor properties defined in column objects must be included in each data object.

Example:

```ts
type Data = {
	name: string;
	email: string;
	age: number;
};

const data = React.useMemo<Data[]>(
	() => [
		{
			name: 'Sergio',
			email: 'sergiogcosgaya@gmail.com',
			age: new Date()
		},
		{
			name: 'Ruben',
			email: 'ruben@fakemail.com',
			age: new Date()
		}
	],
	[]
);
```

### `tableOptions`

The options prop can be used to specify other optional `react-table` options than `columns` and `data`. All options passed will be directly passed to `useTable` hook.

Important:

- Use it if you want to control from outside some features as sorting or pagination.
- Remember to check additional options added by each available plugin!

### `rowsCount`

When controlling from outside (see below), use this prop to specify the total number of rows that the table is using. If this prop is not provided, an approximated calculation using `pageSize` and `pageCount` props will be used.

## Columns width

By default, each column width **is automatically defined based on all cells value**.

### Automatic width

The Table component computes each column width based on the cell value obtained from the data. There are some considerations:

- The minimum width is 20.
- The maximum width is 400.
- Remember to set a min-width in the Table component to avoid columns to be positioned over another.
- Use the `getCellWidthText` column option to specify a different text value than the provided in the data:

  Sometimes the data value and the content shown in the cell don't match, this will cause the Table to not compute the width correctly because it is using an incorrect string. To solve this problem, you can use the column option **`getCellWidthText`** which receives the table row object and must return a new string.
  **This returned string will be used in order to compute the column width.**

### Custom width

You can use custom widths using the prop `width` to set flex components width. Keep in mind that **its values are flex-basis values** and not pixels. For example, if you want a column to be the double than another, you can set widths of `50` and `100`. Using the `width` column option will prevent that column from performing the automatic width computation.

TIP: Use both `minWidth` and `maxWidth` values to define a **fixed column width**.

## Controlled vs uncontrolled

The Table component can be used in both controlled or uncontrolled modes. Keep clear when to use each mode.

_IMPORTANT_: Remember that filtering is always controlled from outside!

#### _Controlled_

Use it when parent component **IS RESPONSIBLE** for updating the data to be shown in the table. When controlling from outside, you should use `manualPagination`, `manualSortBy` and other table props. When using these props, the table won't perform sorting, pagination nor ordering by its own and it **PRINTS ALL ROWS PROVIDED IN `data`** prop, so only specific sorted, paginated and filtered rows should be provided.

The parent component is responsible to listen for user interactions using handler props. There are many handlers that can be used to listen for changes in pagination or sorting, but **BY DEFAULT the prop `onSettingsChange` SHOULD BE USED** to listen for these changes!

#### _Uncontrolled_

Use the uncontrolled mode when you want let the table the responsibility for sorting and paginate all data. In this mode, you only need to provide the data and let the table do the magic.

## Sorting

The Table component uses the [useSortBy](https://react-table.tanstack.com/docs/api/useSortBy) plugin to easily enable some sorting features.

- Multi-sorting is not compatible at this moment.
- Sorting is enabled by default. You can disable it using the `disableSortBy` table option.
- By default, all columns are sortable by default. Disable it using the `disableSortBy` column option.
- If the column is sortable, the table can be sorted by clicking on a column header.
- You can use a pre-defined sort function (see docs) or use a custom one passing a function to `sortType` column option.
  **IMPORTANT**: Custom sorting functions can be added as pre-defined functions in Table component. If it is a common use case, add the sorting function in Table level please.

If you want to control the sorting from outside the table, set the option `manualSortBy` with a `true` value. Then, use the `onSortChange` prop to handle the changes.

Custom pre-defined sorting options:

- `boolean`: Sorts columns based on boolean values.

## Filtering

The Table component uses the [useFilters](https://react-table.tanstack.com/docs/api/useFilters) and [useGlobalFilter](https://react-table.tanstack.com/docs/api/useGlobalFilter) plugins to easily enable some filtering features.

**IMPORTANT**: You can follow the same approach implementing the filters inside the Table if wanted but the `Table` component has been implemented to be fully controlled from outside using the `filters` prop.

- Use `filters` and `globalFilter` props to modify the filters to be used.
- Don't use `tableOptions.initialState.filters` option. Use `filters` prop instead. By default, in first render the `tableOptions.initialState.filters` is used if passed.
- Don't use `tableOptions.initialState.globalFilter` option. Use `globalFilter` prop instead. By default, in first render the `tableOptions.initialState.globalFilter` is used if passed.
- To prevent a column to be filtered using the `globalFilter` use the `disableGlobalFilter` column option.
- Remember to specify a `filter` column option and/or `globalFilter` table option to specify filter type. A function can be used for custom filtering. If string is passed, a pre-defined or built-in filter is used.
  **IMPORTANT**: Custom filtering functions can be added as pre-defined functions in Table component. If it is a common use case, add the sorting function in Table level please.

_Controlled vs uncontrolled filtering:_

The above docs should be used to filter the data provided with the `data` prop. This means the filters will be applied into provided `data`. **If you want to fully control the filtering from outside, DO NOT USE** these filters features but provide the data filtered instead.

## Pagination

The Table component uses the [usePagination](https://react-table.tanstack.com/docs/api/usePagination) plugin in order to easily enable pagination features.

- Use the `Table.Pagination` component to show the built-in pagination component.
- Use the `onSettingsChange` prop in `Table` or `onGoToPage` prop in `Table.Pagination` to handle pagination user actions.
- You can control from outside the current page as its size by changing `pageIndex`, `pageSize` and `pageCount` props.
- To use a full controlled mode use the `manualPagination` table option.

_Controlled vs uncontrolled pagination:_

When using the `manualPagination` prop the table will still change between pages but it won't perform any pagination inside data, showing in the table all the data provided! When using it, the parent component **should only provide the data for the current paginated page**.
