# Tabs

A component used to render tabs to organize content across different views, and where only one view can be visible at a time.

When the list of tab exceeds the width of the Tabs component , the user must scroll to the right to see the next ones.

The active tab may be set using the `defaultTab` prop.

### Tabs.Header vs Tabs.Menu

The reason for having two nested `Box` like this relapses on the different kind of situations and designs that `Tabs` can have.

By default the `Tabs.Header` will come with a border on the bottom. So it's easy to identify it.

You can play with those `Box` depending if you need a fluid content or inside a container.

<Story
	name="playground"
	args={{
		defaultTab: 'tab_1',
		width: '100%',
		maxWidth: '100%'
	}}
	parameters={getPlaygroundStoryParams()}
>
	{args => (
		<Tabs {...args}>
			<Tabs.Header>
				<Tabs.Menu>
					{['tab_1', 'tab_2', 'tab_3', 'tab_4', 'tab_5', 'tab_6', 'tab_7', 'tab_8'].map((tab_id, index) => (
						<Tabs.Tab key={`${tab_id}`} id={`${tab_id}`}>
							{`${tab_id}`}
						</Tabs.Tab>
					))}
				</Tabs.Menu>
			</Tabs.Header>
			{['tab_1', 'tab_2', 'tab_3', 'tab_4', 'tab_5', 'tab_6', 'tab_7', 'tab_8'].map((tab_id, index) => (
				<Tabs.Content key={`${tab_id}`} id={`${tab_id}`} px={3} height={200}>
					This is the content of {`${tab_id}`}
				</Tabs.Content>
			))}
		</Tabs>
	)}
</Story>

## Default state

By default, every tab has `hover` state except the active one.

<Canvas>
	<Story name="default" title="default" parameters={getFixedStoryParams()}>
		<Tabs defaultTab="tab_1" mt={3}>
			<Tabs.Header>
				<Tabs.Menu>

					<Tabs.Tab id="tab_2">Tab 2</Tabs.Tab>
				</Tabs.Menu>
			</Tabs.Header>
			<Tabs.Content id="tab_1" px={3} height={200}>
				This is the content of Tab 1
			</Tabs.Content>
			<Tabs.Content id="tab_2" px={3} height={200}>
				This is the content of Tab 2
			</Tabs.Content>
		</Tabs>
	</Story>
</Canvas>

## With error

You can add the `isError` prop to a tab, to hightlight errors.

<Canvas>
	<Story name="error" title="error" parameters={getFixedStoryParams()}>
		<Tabs defaultTab="tab_1" mt={3}>
			<Tabs.Header>
				<Tabs.Menu>
					<Tabs.Tab isError id="tab_1">
						Tab 1
					</Tabs.Tab>
				</Tabs.Menu>
			</Tabs.Header>
			<Tabs.Content id="tab_1" px={3} height={200}>
				This is the content of Tab 1
			</Tabs.Content>
		</Tabs>
	</Story>
</Canvas>

## Tabs with counter

You can use Counter to show errors in a tab, inside a form, for example.

<Canvas>
	<Story name="with counter" title="with counter" parameters={getFixedStoryParams()}>
		<Tabs defaultTab="description" width={500}>
			<Tabs.Header>
				<Tabs.Menu>
					<Tabs.Tab id="description">Description</Tabs.Tab>
					<Tabs.Tab id="settings" isError>
						Settings
						<Counter aspectSize="s" ml={2} numberOfItems={5} variant="red" />
					</Tabs.Tab>
				</Tabs.Menu>
			</Tabs.Header>
			<Tabs.Content id="description" px={3} height={200}>
				This is the content of description tab.
			</Tabs.Content>
			<Tabs.Content id="settings" px={3} height={200}>
				This is the content of settings tab with errors inside.
			</Tabs.Content>
		</Tabs>
	</Story>
</Canvas>

## Tabs with icon

You can use Icon and Status to show errors in a tab, inside a form, for example. Useful in mobile views.

	</Story>
</Canvas>
