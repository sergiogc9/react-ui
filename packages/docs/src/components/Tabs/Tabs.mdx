# Tabs

A component used to render tabs to organize content across different views, and where only one view can be visible at a time.

When the list of tab exceeds the width of the Tabs component , the user must scroll to the right to see the next ones.

The active tab may be set using the `defaultTab` prop.

### Tabs.Header vs Tabs.Menu

The reason for having two nested `Box` like this relapses on the different kind of situations and designs that `Tabs` can have.

By default the `Tabs.Header` will come with a border on the bottom. So it's easy to identify it.

You can play with those `Box` depending if you need a fluid content or inside a container.

## Tabs menu layout

The tabs items in the menu can have diferent sizes and layouts depending on the `tabsLayout` prop available in the `Tabs` component. The options are:

### `big-evenly`:

The tabs width are the same distributed evenly through all tabs. This is the default.

<Canvas of={TabsMeta.BigEvenly} />

### `small-left`:

The tabs width depends on the content and they're place on the left.

<Canvas of={TabsMeta.SmallLeft} />

### `small-evenly`:

The tabs width depends on the content and they're placed evenly.

<Canvas of={TabsMeta.SmallEvenly} />
